Local-first CI/CD · Apple Silicon · Metal receipts

GitHub triggers. Your Mac proves.

This is a new product loop for native Mac utilities: GitHub Actions becomes the control plane, but your Apple Silicon Mac performs the DMG build, hardware telemetry, and future Metal/MLX policy training locally.

Pipeline

workflow_dispatch → self-hosted macOS ARM64 runner → compile guardian.mm → build AI-RAM-Guardian.dmg → collect safe Metal/hardware receipt → upload DMG + sha256 + receipt artifact

Why local?

GitHub-hosted Macs can build software. They cannot observe your real Devin, Windsurf, Claude, Cursor, and Metal workload. A self-hosted runner turns your actual machine into the lab without sending private documents or prompts to the cloud.

What gets collected?

Only hardware/build telemetry: machine model, OS version, memory size, Metal/GPU summary, build hash, and explicit Guardian receipts. No chats, prompts, browser data, documents, secrets, or app content.

Future Metal/ML loop

The valuable dataset is machine behavior, not private content: available memory, pressure trend, thermal state, helper RSS, intervention type, before/after deltas, build version, and hardware profile hash. That supports small on-device models: Kalman pressure filters, learned anomaly baselines, local policy tuning, and release regression detection.
